☑️ **Задача-1 – приглушить hover-эффект карточки**

Сейчас `hover:bg-white/10` (или эквивалент) подсветка делает карточку почти белой и «слепит».  
Нужно: легкий, еле-заметный fade, одинаковый для light/dark.

Что делаем  
1. В файле Card / CurriculumCard (где сама карточка) заменяем класс «bg-*» для `hover:`:  
   • Light theme → `hover:bg-slate-100`  
   • Dark theme  → `hover:bg-white/5`  
2. Плавность: `transition-colors duration-150`.  
3. Проверить: при наведении светлая тема даёт +5-7 % яркости, тёмная – едва-серый оверлей, текст остаётся читаемым, кнопки «Редактировать / Удалить» не двигаются.

☑️ **Задача-2 – правки титульного листа теряются при переходе**

Симптом: меняю «Форма обучения», «Уровень образования» или «Срок обучения», кликаю на таб «График» / ухожу со страницы → возвращаюсь – старые значения.

Причина  
• `EditCurriculumPlan` хранит форму в локальном state; переход на другой таб размонтирует компонент, state теряется, а `useAutoSave` вызов для титула не сделан.

Что делаем  
1. **Единый объект planForm** в родителе `EditCurriculumPlan` (useState).  
   • Передаём его и setter вниз в `TitleTab`, `GraphTab`, т.д.  
   • Таким образом вкладки редактируют один и тот же state.

2. **savePlan()**  
   • При нажатии «Сохранить изменения» – PATCH `/api/curriculum/plans/:id` с полями из planForm.  
   • После успешного ответа сбрасываем `isDirty=false`.

3. **Автосохранение только для Graph/Weeks**  
   • `useAutoSave` остаётся на таблице недель.  
   • Для титульного листа НЕ вызываем autosave, чтобы не спамить запросами.

4. **Guard при переключении вкладок**  
   • Если `isDirty===true` – показываем простой `confirm("Сохранить изменения титульного листа?")`.  
   • На «OK» → `await savePlan()` ; на «Cancel» → откат к последнему сохранённому (state из бекенда).

5. Проверка  
   • Меняем «Форма обучения»       → переходим на «График» → диалог → «OK» → данные сохраняются, при возврате – всё на месте.  
   • Меняем, жмём «Cancel» – возвращается старое значение.  
   • Hover карточки стал мягким.

После выполнения — короткий отчёт + скрин «до/после» hover, и видео/гиф с переходом вкладок, где правки не теряются.
